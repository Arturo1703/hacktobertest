Lets learn a bit about machine learning using a very high-level library , called ml5 by Danniel Siffman.

In these brief tutorial we will be doing some examples on these library which is on top of tensorflow.js

This library uses p5.js and pre-trained models to abstract from the mathematics of machine learning. If you want to feel the pain of learning the math, ask me for resources.

Exercise #3

The harder custom made classifier

You may have noticed the difference of complexity from the first exercise to the second one. Now we are moving on to the more heavy duty exercises.

We are going to be teaching the pre-trained model to learn about our own images!. This technique is called transfer model. Its like re-training an existing model without giving the model the labels.

The code you are given as a starting point (it is a slightly modified version from the first exercise) , can be found in:https://github.com/dlgeraghty/hacktobertest/tree/master/learning-ml-5/webcam-image-classification The given code is a simplified version of what you have to do.

The goal of this exercise is to re-train the model in order to distinguish between a happy and a sad face!

Here is where things start to get developmentwise more serious, if you accept the challenge, all I can give you is the documentation page: https://ml5js.org/reference/api-FeatureExtractor/

More detail in the code you are give, there is a pseudocode there you can use

Exercise #4

The study-code-sleep-repeat project

This is an exercise that is to be done by the public who wants to get involved in a mini community

The idea, is to build a face-recognition service that works as a lock, with the knowledge you have learned in these exercises.

Some thing you may want to take into consideration are:
  • How do I get sufficient variety in the samples?
  • Do I have to re-train the model each time I want to use it?
  • What is the level of accuracy that I can get?
  • May I consider learning a lower-level technology such as tensorflow to improve performace?
All the people who worked on this talk and the community in general would be very happy and welcoming for a project like this. If some of you happen to want to start it, please let us know!
